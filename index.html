Heute lernen wir Git & Github etwas besser kennen

  1. lokales Git-Repo erstellen ==> git init
  2. Wie bereite ich Dateien vor ?
    1. alle dateien/änderungen für die stating area vorbereiten ==> git add .
    2. bestimmte datei/änderung für die stating area vorbereiten ==> git add "datei.name"
  3. stating area in das lokale git repo speichern ==> git commit -m "bitte etwas aussagekräftiges"
  4. verbindung zwischen lokalem git repo und github repo erstellen ==> git remote add origin "git@github.com:LinkZumGithubRepo"
  5. einen main branch erstellen ==> git branch -M main
  6. lokales git repo hochladen ins github repo ==> git push -u origin main

!!! Schritt 4 bis 6 bekommt ihr von GitHub nach erstellen eines Repositories dort !!!

Was ist ein branch ?

Ein Branch ist ein seperates Arbeitsverzeichnis, die unabhängig voneinander existieren und bearbeitet werden können. Wir haben aktuell nur mit dem main branch gearbeitet.

Was ist ein repository ?

Repository = Depot/Lager, hier werde alle von uns hinzugefügen Dateien gespeichert, um sie später gebündelt zu haben.

Was ist die stating area ?

Ist wie ein Wartebereich beim Arzt, hier werden die von uns geänderten Dateien vorbereitet um ins Repository zugelangen
